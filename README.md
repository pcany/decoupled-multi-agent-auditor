## Executive Summary & Core Value

Modern enterprise and banking environments are rapidly adopting generative AI agents. However, standard security layers fail to address the **"Rationalization Problem"**: over long-term context windows (chats spanning months or years), a helpful agent can be subtly manipulated through social engineering (the Salami-Tactic). Because LLMs optimize for user helpfulness, they actively begin to justify and normalize anomalous user behavior within the conversation log, blinding static filters and standard guardrails.

**The Solution:** The *Decoupled Multi-Agent Auditor Framework (DMAF)* introduces a strict, decoupled supervision architecture. By separating the execution layer from the agent's reasoning path and employing an independent, adversarially instructed **Auditor Agent**, DMAF detects systemic drifts in intent before irreversible actions are triggered. 

This framework transitions AI safety from vulnerable text-surface filters to **uncompromised system boundaries** and deterministic metric tracking.

## 🛠 Key Course Concepts Implemented

To satisfy the requirements of the Kaggle AI Agents Intensive Capstone, this production-ready repository integrates three core paradigms:

1. **Multi-Agent System & Orchestration (ADK):** Implemented via a stateful asynchronous architecture using the official Agent Development Kit. It orchestrates a cooperative yet critical loop between the primary user-facing agent and the monitoring supervisor.
2. **Security Features (Inline Auditing):** Solves the unaddressed challenge of context poisoning. The system isolates the agent's intent vector, monitors behavioral entropy, and provides an independent meta-analysis of the user's historical cumulative actions.
3. **Model Context Protocol (MCP) Server Integration:** The Auditor Agent relies on a strict MCP server layer to securely access and audit historical conversation logs and execute deterministic statistical analysis without risking data leakages or credential exposure.

## 📊 Core Analytical Pillars (How It Works)

*   **Semantic Divergence Engine:** The Auditor completely strips away the polite phrasing or excuses generated by the primary agent and reviews only the raw, cumulative physical capabilities granted to the user over time.
*   **Behavioral Entropy Tracking:** Tracks the statistical variance of safety disclaimers and critical friction points. When a primary agent becomes "brainwashed" (collusion), its entropy drops, signaling the system to alert compliance.
*   **Deterministic Safety Isolation:** No agent has direct write access. Every proposed action is emitted as a strictly defined JSON payload, which is parsed and verified by a non-AI system layer against immutable enterprise whitelists.

### 📝 Project Journey & Practitioner's Note
This project is built from the ground up to reflect the rigid constraints of real-world, highly regulated software engineering. In production environments (such as banking-grade infrastructure), we cannot rely on the "good behavior" or alignment of an LLM.

By designing a decoupled system where the code, not the model, dictates the ultimate security boundaries, DMAF demonstrates how enterprises can confidently harvest the efficiency of autonomous agents without failing regulatory compliance or succumbing to long-term adversarial prompt engineering.
